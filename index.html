Introduction

Why should I have my own website?

There are several reasons you may want a personal website; whether you want a blog, [[portfolio]], cv, or just a place to tell the world about yourself outside of traditional social media, a personal website is a great thing to have. Having a personal website does not require that you host that site yourself, and in fact most people don’t host hteir own sites - many services such as WordPress, Squarespace, and Wix.com offer people a fast, easy, and affordable way to build and host a website, and for most people, a service like this is probably the best option.

Why would I want to host the site myself?

Realistically, most people probably don’t. This guide is specifically targeted at people who wish to better understand how the web works by building a small piece of it themselves, and while this route can offer a site that is lighter, faster, and gives you more granular control than those built with the management systems mentioned above, the extra effort probably isn’t worth it unless you have a strong desire to learn and tinker. For those who are willing to put in the effort, however, hosting your own website can be incredibly rewarding and allows you to build something truly unique without the limitations of hosting platforms.

What is a static website and why might I want one?

Simply put, a static website is a website with fixed content built using [[HTML]], [[CSS]], and possibly [[JavaScript]]. The alternative to a static website is a dynamic website, which is what you’re probably used to seeing with sites like Twitter, Google, and Netflix. These sites are built with web frameworks like React or Angular and dynamically serve content to users, which is why the content you see is different every time you visit one of these sites. It is important to note that static websites can still be responsive, cross-platform, and can even have dynamic elements (though that’s outside the scope of this guide). The reason this guide focuses on static sites, aside from their relative simplicity, is because they provide a strong, stable foundation on which to build, tinker, and ultimately deepen your understanding of the web.

What should I already know?

This guide assumes that you have no experience with the tools and platforms used. With that in mind, you should be very comfortable using a computer, and you mustn’t fear the [[command line]] - it is a powerful tool that makes everything in this guide possible. Because this guide focuses on [[hosting]] rather than designing a website, there won’t be an explanation of how to build your website with HTML and CSS; that said, there will be links to external resources for that at the end of the guide.

How long will this take?

Due to the nature of wireless networking, wait times between certain steps can be unpredictable. You should expect to spend about two to three hours getting your site set up (less if you’re already familiar with the tools), and many more hours tinkering and troubleshooting as you expand your site into whatever you want it to be.

[[Sentence Definitions]] ### Important Technologies & Terminology - [[HTTP]] (Hypertext Transfer Protocol) - HTTP is an internet protocol that delivers websites to web browsers. - [[SSH]] (Secure Shell Protocol) - SSH is a networking protocol that allows users to access a server remotely. - VPS (virtual private server) - A VPS is a cloud server accessible remotely that can run the software required to host a website. - [[DigitalOcean]] - DigitalOcean is a cloud service provider that offers virtual private servers (called droplets) for a monthly fee. There are many alternatives available, including Linode and OrangeWebsite. - [[Nginx]] (pronounced engine-x) - Nginx is a HTTP server, or a piece of software that delivers a website to a visitor’s browser. - [[Linux]] - Linux is an operating system that is used to run most web servers. - [[Bash]] - Bash is a command line interpreter that is used to execute instructions in Linux. - [[HTML]] (HyperText Markup Language) - HTML is a markup language used to create documents meant for the web. - [[CSS]] (Cascading Style Sheets) - CSS is a style language used to layout the visual design of HTML documents. - [[JavaScript]] - JavaScript is a scripting language that runs in web browsers and is used to make websites more dynamic. - Domain Name - A domain name is a unique identifier for a website that allows it to be easily accessed via a web browser. (E.g., example.com, wikipedia.org) - Subdomain - A subdomain is a string of text preceding a website’s main domain name. (E.g., www.example.com, en.wikipedia.org) - DNS (Domain Name System) - DNS is the naming system that manages domain names and links them to web servers. - Symbolic Link - A symbolic link is a file that points to a different file or directory, similar to shortcuts in Windows. - TLS (Transport Layer Security) - TLS is a security protocol that encrypts data sent to websites to protect user privacy. - Certbot - Certbot is a command-line tool created by the Electronic Frontier Foundation to simplify the process of setting up TLS for a website.

Warning!

This guide makes heavy use of bash, and specifically the sudo command, which allows the user to run commands with elevated privileges. When following any guide involving bash it is essential that you carefully check any command involving sudo before executing, as this command allows you to do significant damage to your system if used improperly.

Caution

While this guide will follow basic security practices, there are many security risks that come with hosting a website. You should never upload anything to the web server that you wouldn’t want publicly visible, and you shouldn’t attempt to handle user data without significant further research.

Overview

These instructions will involve the following actions: 1. Set up ssh on your device 2. Sign up for DigitalOcean and create a droplet 3. Install web hosting software on server 4. Set up a domain name for your site 5. Configure website on server 6. Set up TLS 7. Upload Custom Website files

What You’ll Need

The only things you’ll need to get started are a computer connected to the internet and about $15. This guide will primarily target Windows users, but everything here is possible on nearly any modern operating system. Over the course of this guide, you’ll have to pay about $5 per month for a server, and about $10 per year for a domain name.

  1. Sign up for DigitalOcean.
    1. Go to www.digitalocean.com
    2. Click “Sign Up”
    3. Follow the site’s instructions to create your account.
      • Note: You will be required to enter payment information and choose a project name
      • Once your account has been created, you should see the Dashboard in Figure 4.

  2. Set up a droplet.
    • [Figure 5: Droplet creation drop-down]
    1. Click the green Create button on the top right of the dashboard (see Figure 5).
    2. Click Droplets in the drop-down menu that appears.
    3. Choose options for your droplet based on the following guidelines:
      • Region: Choose the region closest to you for ideal performance.

  2. Set up user account.
    1. Choose a username that only contains lowercase letters, with no spaces or special characters.

      • adduser username

Website Configuration

  1. Create website files on server.
    1. Run the following command to create a directory for your website.
      • sudo mkdir -p /var/www/[mydomain]/html
    2. Grant your user ownership of the directory with the following command.
      • sudo chown -R $USER:$USER /var/www/[mydomain]/html
    3. Run the following command to create an empty html document using the text editor nano.
      • sudo nano /var/www/[mydomain]/html/index.html
    4. Enter the following sample html into the document, or create your own. html <html> <head> <title>My Personal Website</title> </head> <body> <h1>My website is working.</h1> </body> </html>
    5. Press Ctrl+x to exit.
    6. Press y to save the file.
    7. Press enter to keep the default filename.
  2. Setup Nginx configuration.

    1. Create and open a configuration file with the following command.

      • sudo nano /etc/nginx/sites-available/[mydomain]

    2. Enter the following text in the file, making sure to replace [mydomain] with your domain name. ``` server { listen 80; listen [::]:80;

       root /var/www/[mydomain]/html;
 index index.html index.htm;

 server_name [mydomain] www.[mydomain];

 location / {
     try_files $uri $uri/ =404;
 }

      } ```

    3. Run the following command to create a symbolic link that enables your website.

      • sudo ln -s /etc/nginx/sites-available/[mydomain] /etc/nginx/sites-enabled/

    4. Disable the default website for Nginx with the following command:

      • sudo rm /etc/nginx/sites-enabled/default

    5. Test your Nginx configuration for errors with the following command.

      • sudo nginx -t
      • If there are no errors, you should see the following output:
      nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful
      • Note: If you do see errors, you’ll have to open the previous file again and fix the mistakes mentioned in the command output.

    6. Restart Nginx with the following command.

      • sudo systemctl restart nginx
      • There will be no output if the command runs successfully.

    7. Type your domain name in the address bar of your web browser.

      • You should see the page shown in Figure 11, with some differences if you made your own html earlier.
    • [Figure 11: Custom website]

TLS Setup

  1. Install Certbot
    1. Run the following command to ensure the package installer snapd is up to date.
      • sudo snap install core; sudo snap refresh core
      • When the installation is done, you should see the following output.
      core 16-2.58.3 from Canonical✓ installed
snap "core" has no updates available
    2. Install Certbot using snapd with the following command:
      • sudo snap install --classic certbot
      • The output of the command should read certbot 2.4.0 from Certbot Project (certbot-eff✓) installed
    3. Create a symbolic link for Certbot with the following command:
      • sudo ln -s /snap/bin/certbot /usr/bin/certbot
  2. Run Certbot.
    1. Use the following command to run Certbot:
      • sudo certbot --nginx
    2. Enter your email address when prompted
    3. Enter y to agree to Certbot’s terms of service.
    4. Enter either y for yes or n for no when asked whether to share your email with the EFF.
    5. Press enter to activate HTTPS for your domain.
      • You should see the following line in your output:
      • Congratulations! You have successfully enabled HTTPS on https://[mydomain] and https://www.[mydomain]
  3. Test HTTPS.
    • navigate to https://[mydomain] in your browser.
      • You should see a lock symbol next to your website name indicating that the site is secure (Figure 12)
    • [Figure 12]

Website Customization

  • Note: While designing a website is outside the scope of this guide, once you do make a website you’ll have to upload the files to your server. There are resources linked at the end of this guide to help you design your website, and you can come back to this section whenever you have files you need to upload.
  1. (On Windows) Open Windows file explorer.
  2. Navigate to folder with your website files.
  3. Click the title bar to select the directory where your files are located (Figure 13).
  • [Figure 13]
  1. Copy the directory to your clipboard.
  2. Run the following command to copy the files to the server, replacing [directory] with the directory you copied.
    • scp [directory]/* [username]@[ip-address]:/var/www/[mydomain]/html/
    • Note: Any files already on the server with the same name as files you upload will be overwritten.
    • Note: The first page users see when they visit your website will always be the file titled index.html
    • If you go to your website in a web browser, you should see the changes you made. If they don’t appear right away, you may need to find instructions to clear the cache of whatever browser you’re using.
